## 2.Create queries with NoSQL
* returns all fields from a container ``` SELECT * FROM products ```
* returns only a few fields ``` SELECT products.id, products.name, products.price, products.categoryName FROM products ```
* any name can use ``` SELECT p.name, p.price FROM p ```
* use any word or phrase ``` SELECT califragilisticexpialidocious.id, califragilisticexpialidocious.categoryId FROM califragilisticexpialidocious ```
* use alias ``` SELECT alternativealias.id, alternativealias.name FROM reallyinterestingdatasource alternativealias ```
* filter by WHERE ``` SELECT p.name, p.categoryName, p.price FROM products p WHERE p.price >= 50 AND p.price <= 100 ```

* IS_NUMBER function -  Some persist pricing information using string data, while others may store pricing information using numbers.

## 5.Execute queries in the SDK
* Microsoft.Azure.Cosmos.Container class - create a query, issue the query to Azure Cosmos DB for NoSQL, set up an asynchronous stream in C#, and return items efficiently back to the client.
* c# returns all products:
    ```
    QueryDefinition query = new ("SELECT * FROM products p");
    public class Product
    {
        public string id { get; set; }
        public string name { get; set; }
        public string price { get; set; }
    }
    
    // GetItemQueryIterator - await foreach loop, automatically handles the looping and pagination to go to the server and get each subsequent page of results.
    //Within the foreach loop, add your code to handle each item; in this example, each item’s id, name, and price is output to the console:
    
    using (FeedIterator<Product> feedIterator = this.Container.GetItemQueryIterator<Product>(
        query,
        null,
        new QueryRequestOptions() { }))
    {
        while (feedIterator.HasMoreResults)
        {
            foreach(var item in await feedIterator.ReadNextAsync())
            {
                Console.WriteLine($"[{item.productid}]\t{item.name,35}\t{item.price,15:C}");
            }
        }
    }
    ```
