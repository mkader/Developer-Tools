## 1. Create cross-product queries
* A JOIN in Azure Cosmos DB for NoSQL is different from a JOIN in a RDMS as its only scope is a single item.
* A JOIN creates a cross-product between different sections of a single item.

* Implement variables in queries
  ```
  string sql = "SELECT p.name, t.name AS tag FROM products p JOIN t IN p.tags WHERE p.price > 500"
  QueryDefinition query = new (sql);
  ```
  ```
  string sql = "SELECT p.name, t.name AS tag FROM products p JOIN t IN p.tags WHERE p.price > @lower"
  QueryDefinition query = new (sql)
      .WithParameter("@lower", 500);
  ```
  ```
  string sql = "SELECT p.name, t.name AS tag FROM products p JOIN t IN p.tags WHERE p.price >= @lower AND p.price <= @upper"
  QueryDefinition query = new (sql)
      .WithParameter("@lower", 500)
      .WithParameter("@upper", 1000);
  ```
  * Benefits of Parameterized Queries
    * Improved Security: Helps prevent SQL injection attacks.
    * Flexibility: Allows you to modify query filters dynamically without changing the SQL query structure.
    * Code Reusability: Makes it easier to reuse query definitions across multiple queries with different parameter values.

## 2.Paginate query results
* MaxItemCount - specify how many items you wish to return for each page of results.
* HasMoreResults boolean property that indicates more pages to return server-side.
* ReadNextAsync method that gets the next set of items into an enumerable collection that can be iterated over with a foreach loop.
    ```
    string sql = "SELECT * FROM products WHERE p.price > 500";
    QueryDefinition query = new (sql);
    QueryRequestOptions options = new()
    {
        MaxItemCount = 100
    };
    
    FeedIterator<Product> iterator = container.GetItemQueryIterator<Product>(query, requestOptions: options);
    
    
    C#
    while(iterator.HasMoreResults)
    {
        foreach(Product product in await iterator.ReadNextAsync())
        {
            // Handle individual items
        }
    }
    ```
